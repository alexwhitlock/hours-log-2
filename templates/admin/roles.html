How this works

Each role has a monthly hours allotment. Members assigned to a role automatically receive a pre-approved hours record on the last day of each month — they don't need to log or submit anything.

Auto-generation: On the last day of each month the server automatically creates one approved record per active assignment. Each assignment can only have one record per calendar month — running twice does nothing.

Pro-ration: If a member's start or end date falls mid-month, their hours for that month are pro-rated based on how many days of the month they were active. For example, a member starting on the 16th of a 30-day month receives 50% of the monthly allotment.

Deduplication: Every auto-generated record is tagged with the assignment that created it. Before generating, the system checks whether a record already exists for that assignment in that month. Manual records are never affected.

Generate Now backfills any past months (from the member's start date up to last month) that are missing a record, with pro-ration applied to partial months. It does not touch the current month — that will be created automatically at month end. Safe to click multiple times.
